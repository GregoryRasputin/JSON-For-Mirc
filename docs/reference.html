About

The following is a reference to all commands and identifiers provided by JSON For mIRC. To understand this documentation you must first understand the following:

Parameters

Most of the commands and identifiers require some form of input, that input is refered to as a parameter. All parameters in these documents begin with @ and are explained in the command's or identifier's section that lists it.

Different commands or identifers may use the same name for a parameter but each parameter's meaning and requirements are unique to the command or identifier that specified it

Switches

If the first parameter of a command begins with - it is a switch. Switches alter how the command processes the input or behaves after the input has been processed.

Unless otherwise stated all switches are optional and should only be used to enact the alternative processing.

Properties

Similar to command switches, identifier properties alter how the inputs are processed or how the identifier behaves. Properties are prefixed to the identifer via a . and are completely optional.

/JSONList
Lists all open JSON handlers
Switches
None
Parameters
None
/JSONOpen -dbfuUw @Name @Input
Attempts to create a JSON handler
Switches
d
Closes the JSON handler and all children when the script block exits
b

@Input is the name of a bvar containing the JSON to parse.

This switch cannot be used with the f u or U switches

f

@Input is a path to a file containing the JSON to parse.

This switch cannot be used with the b u or U switches

u

@Input is a URL from which the JSON to parse should be retrieved.

This switch cannot be used with the b f or U switches

U

@Input is a URL from which data should be retrieve but NOT parsed. This allows for the json http handling to be used as a simple web-request client

This switch cannot be used with the b f or u switches

w

The URL request should wait until /JSONHttpFetch is called. This is to allow the request method and headers to be set.

This switch can only be used if the u or U switch is specified

Parameters
@Name
Required

The name in which the handler will be refered to

Cannot contain the characters: ? * : or space

Cannot be a numerical value

@Input
Required

The input JSON data to be parsed

if the b switch is specified the input will be considered a bvar

if the f switch is specified the input will be considered a file

if the u or U switch is specified the input will be considered a URL

/JSONHttpFetch -bf @Name @Data
Performs a JSON handler's pending HTTP request

To use this command, the handler must have been created with the w switches and the HTTP request must be pending.

Switches
b

@Data is the name of a bvar containing the JSON to parse.

This switch cannot be used with the f switch

f

@Data is a path to a file containing the JSON to parse.

This switch cannot be used with the b switch

Parameters
@Name
Required

The name of an open JSON handler with a pending HTTP request

@Data
Optional

The data to send with the HTTP request

If the b switch is specified, the data will be consider a bvar

If the f switch is specified, the data will be consider a file path

$JSON(@Name)
Returns the name of the JSON handle if it exists
Parameters
@Name
Required

The name of the JSON handle to look up

If a numerical value is given, the name of the Nth open JSON handle is returned. If 0 then the total number of open JSON handles is returned

$JSON(@Name|Ref|N, @Members...)
Walks to the JSON handle using the specified member(s) and returns a reference to the referenced member
Parameters
@Name|Ref
Required

The name of the JSON handle to walk.

Can be a reference returned by a previous $JSON() call.

If a numerical value is given, the Nth open JSON handle will be walked

@Members...
Required

One or more members to walk though in the JSON

To walk through multiple members add each as an extra argument when calling $JSON().

$JSON(@Name|Ref|N, @Members...) 'Fuzzy' Property-Prefix
Uses Fuzzy lookup to reference members

All properties of which walk the JSON can be prefixed with fuzzy to indicate fuzzy lookup should be used.

Fuzzy lookups take place on JSON Objects where referencing integers are considered indexes instead of keys, and case-sensitivity within specified keys is ignored

Parameters
@Name|Ref|N
Required

The JSON handle name, reference or index to walk using fuzzy lookup

@Members...
Required

One or more members to walk through in the JSON data

To walk through multiple members add each as an extra argument when calling $JSONForEach().

Only members prefixed with ~ that would be used in referencing an object's children are assumed to be fuzzy members.

A normal lookup can be forced on members by prefixing the member with =

$JSON() 'ToBvar' Property-Suffix
Stores the result of the $JSON() call in a bvar and then returns that bvar

Unless otherwise stated, all $JSON() props can be suffixed with toBvar.

This suffix is not compatible with the ToFile prop suffix.

$JSON() 'ToFile' Property-Suffix
Writes the call output to a specified file

Unless otherwise stated all $JSON() props can be suffixed with toFile.

This suffix is not compatible with the ToBVar suffix.

Parameters
@File
required

The file to write to.

The path to the file must be writable and the file must NOT exist.

Specified as the second parameter to the $JSON() call directly after the @Name|Ref|N parameter.

$JSONForEach(@Name|Ref|N, @Callback, @ChildMembers...)
Loops over each item in the reference and calls the specified callback with a reference to the child member
Parameters
@Name|Ref|N
Required

The JSON handle name, reference or index to reference.

@Callback
Required

The command to call with each item being looped over

The callback is passed a reference to the child member of which can be used with $JSON() to return various data related to the item.

@ChildMembers...
Optional

One or more members to walk in each looped item before calling the callback with a reference.

To walk through multiple members add each as an extra argument when calling $JSONForEach().

Props
.fuzzy

Uses fuzzy lookups when walking each item being looped over.
