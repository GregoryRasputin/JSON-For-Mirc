About

The following is a reference to all commands and identifiers provided by JSON For mIRC. To understand this documentation you must first understand the following:

Parameters

Most of the commands and identifiers require some form of input, that input is refered to as a parameter. All parameters in these documents begin with @ and are explained in the command's or identifier's section that lists it.

Different commands or identifers may use the same name for a parameter but each parameter's meaning and requirements are unique to the command or identifier that specified it

Switches

If the first parameter of a command begins with - it is a switch. Switches alter how the command processes the input or behaves after the input has been processed.

Unless otherwise stated all switches are optional and should only be used to enact the alternative processing.

Properties

Similar to command switches, identifier properties alter how the inputs are processed or how the identifier behaves. Properties are prefixed to the identifer via a . and are completely optional.

/JSONOpen -dbfuw @Name @Input
Attempts to create a JSON handler
Switches
To Do
Parameters

Example
Creates the JSON handler 'example' where the input data is "some data". The handler will be closed after the script finishes executing
JSONOpen -d example "some data"

Creates the JSON handler 'example' where the input is stored in &bvar
JSONOpen -db example &bvar

Creates the JSON handler 'example' where the input is stored in file.txt
JSONOpen -df example file.txt

;; Creates the JSON handler 'example' where the input is to be retrieved from http://example.com/data.json
JSONOpen -du example http://example.com/data.json

;; Creates the JSON handler 'example' where the input is to be retrieved from http://example.com/data.json. The connection is to wait for /JSONHttpFetch is called
JSONOpen -duw example http://example.com/data.json

/JSONHttpMethod @Name @Method
Sets the method for a JSON handler's pending HTTP request

To use this command, the handler must have been created with the -w switch and the HTTP request must be pending.

Switches
None
Parameters
To Do
Example
Creates the JSON handler 'example' with a pending HTTP request
JSONOpen -duw example http://example.com/data.json

Stores the method for when /JSONHttpFetch is called
JSONHttpMethod example POST

/JSONHttpFetch -bf @Name @Data
Performs a JSON handler's pending HTTP request

To use this command, the handler must have been created with the -w switch and the HTTP request must be pending.

Switches
To Do
Parameters
To Do
Example
Performs the HTTP request and attempts to parse the result
JSONHttpFetch example

Performs the HTTP request sending "some text" with the request and attempts to parse the result
JSONHttpFetch example some text

Performs the HTTP request sending the data in &bvar with the request and attempts to parse the result
JSONHttpFetch -b example &bvar

Performs the HTTP request sending the contents of file.txt with the request and attempts to parse the result
JSONHttpFetch -f example file.txt

$JSON(@Name|Ref|N, @Members) 'Fuzzy' Property-Prefix
Uses Fuzzy lookup to reference members

All properties can be prefixed with fuzzy to indicate fuzzy lookup should be used.

Fuzzy lookups take place on JSON Objects where referencing integers are considered indexes instead of keys, and case-sensitivity within keys is ignored

Parameters
To Do
Example
JSONOpen -d example {"key":"some data"}

Echos: some data
echo -a > $JSON(example, ~0).FuzzyValue
echo -a > $JSON(example, ~KEY).FuzzyValue
$JSON(@Name|Ref|N, @BVar, @Members) 'ToBvar' Property-Suffix
Stores the call output in a specified bvar

All $JSON() properties can be suffixed with ToBvar to indicate the output should be stored in a specified bvar

This suffix is not compatible with the ToFile suffix.

Parameters
To Do
Example
JSONOpen -d example "some data"

Fills &bvar with: some data
noop $JSON(example, &bvar, key).ValueToBvar
$JSON(@Name|Ref|N, @File, @Members) 'ToFile' Property-Suffix
Writes the call output to a specified file

All $JSON() properties can be suffix with ToFile to indicate the output should be wrote to the specified file.

This suffix is not compatible with the ToBVar suffix.

Parameters
To Do
Example
JSONOpen -d example "some data"

Writes to file.txt: some data
noop $JSON(example, file.txt, key).ValueToFile
