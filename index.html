About

JSON For mIRC is a mIRC script that makes use of mIRC's COM interface to parse and access JSON data. Boasting a rich feature set, JSON For mIRC aims to be comprehensive, light-weight and blazingly fast.

History

After scouring the web for a well coded JSON parser, a script by Timi was discovered that made use of mIRC's COM interferace. Timi's JSON parser was functional and fast (compared to other pure mSL implementations) it was found to have many inefficiencies. Due to such, I began to write his own version and JSON For mIRC was born.

The first version brought parsing and accessing but due to poor planning and being built from a piecemeal of personal specialized scripts, the original release had many flaws. As bugs were patched the codebase became harder to manage resulting in an ever increasing difficulty to patch newly discovered bugs or add desired features.

While the project struggled to maintain itself new features were added to mIRC and many coding discoveries were made; this gave rise to a complete rewrite. The rewrite was better planned, held to a higher set of coding guidelines, is open source and version controlled via git/github.

Contribute

If you would like to contribute to the project feel free to fork the repo, edit the files then make a pull request. If the alterations are accepted your pull request will be merged with the master source-code.

You can contribute by doing more than just coding: help is needed with proof reading and phrasing in the documention, helping others learn to use the script, testing alpha/beta releases, and even reporting bugs

What is JSON?

JSON is short for "JavaScript Object Notation"; originally meant for web applications, it has become the go-to data format for many enviornments and platforms. Its structuring makes for small, text-based, outputs that are easily parsable programmatically while still being human legiable.

JSON is NOT a functional data notation. JSON data does not have a means to specify directives(commands) a program must do when attempting to understand the JSON data; the JSON standard strictly prohibits such as a means of ensuring security.

White Space

Previously it was mentioned that JSON is human readable yet most web api's return it as a jumbled mess. This is because JSON intrepreters are required to ignore whitespace so it can be removed without costing data integrity; this is what those APIs do with the intent of reducing network traffic. There are online tools that will re-add whitespace to make the output human readable.

Data Types

Data types are a way to indicate to the intrepreting program what each piece of the data is. JSON has 6 data types: 4 of which are used to represent simple values and 2 for containing data collections.

Primitives

JSON has four primitive data types to represent a single simple value. These data types do not have the complexity or structuring to cantain multiple values the way Container Types do.

Null - Donated by the keyword null

Null is used when there is absolutely no value to reference. That is, the value isn't empty text or 0 or false; there is absolutely nothing to reference.

Boolean - Denoted by the keywords true and false

Boolean is used when a value is meant to only have two states. Its used to say "yes", "true", "on", "no", "false", "off"

Numbers - Denoted by a valid numerical sequence.

Number is used when a value should be intrepreted as a number; Positive numbers, negitive numbers and decimals are all Number data types.

String - Denoted by text wrapped in double quotes: ""

String is used when a value, regardless of its contents should be intrepreted as text. Due to strings being able to contain any text, including double quotes, there are special character sequences that have alternative meanings within the context of a string:

\" - Represents a double quote( " )
\/ - Represents a forward slash( / )
\\ - Represents a back slash( \ )
\f - Represents a form feed character
\r - Represents a carriage-return character
\f - Represents a line-feed character
\t - Represents a tab character
\uNNNN - Represents a unicode character where NNNN is the hexidecimal encoding of the character

Containers

Containers, as their name implies, are used to contain multiple values and can consist of both primitive and other container values.

Array - Denoted by a comma seperated list of values wrapped in brackets: []

Arrays are ordered lists of values where each value is automatically given an index for referencing. Indexes start at zero and increase by one for each value: the first value uses an index of 0, the second an index of 1, the third an index of 2 and so on for each value in the array.

Example: ["a", 1, true, null, ["b", "c"]]

Objects - Donated by a comma seperated list of key-value pairs wrapped in braces: {}

Objects are unordered lists where each value contained is referenced by a given key. All keys are represented as a string and are seperated from their value via the use of a colon(:). Keys are case sensitive so attempting to reference "KEY" will not point to the data contained under "key"

Example: {"key": "value", "key2": true, "key3": [null, "c"]}

Referencing

Referencing specific parts of JSON data is quite simple once you understand the above syntax for JSON data

In the following examples, lines prefixed with ;; are comments, and -> represents referencing an item of a container

;; references 'item' of 'container'
container -> item

Referencing Primitives

If the JSON data is a primitive, it is simply returned by the intrepreter.

json_data = "value";
parsedjson = interpret_JSON(json_data)

;; This would output the text: value
output(parsedjson)

Referencing Container Values

If the JSON data is a container, there's a bit more work involved to access values. A key or index must be specified when accessing the container and successive keys and indexes will need to be specified for nested containers.

json_data = {
    "key1": "value",
    "key2": [
        "index0",
        "index1"
    ],
    "key3": {
        "key3.1": "value2"
    },
    "key4": {
        "key4.1": [
            "value3"
        ]
    }
}

parsedjson = interpret_JSON(json_data)

;; Results in myVar being set to the text: value
myVar = parsedjson -> "key1"

;; Results in myVar being set to the text: index0
;; key2's value is referenced(which is a container)
;; then the 0th indexed item is referenced
myVar = parsedjson -> "key2" -> 0

;; Results in myVar being set to the text: index1
myVar = parsedjson -> "key2" -> 1

;; Results in myVar being set to the text: value2
;; key3's value is referenced(which is a container)
;; then the item with key "key3.1" is referenced
myVar = parsedjson -> "key3" -> "key3.1"

;; Results in myVar being set to the text: value3
;; key4's value is referenced(which is a container)
;; then the value of key4.1 is referenced(another container)
;; then the 0th indexed item is referenced.
myVar = parsedjson -> "key4" -> "key4.1" -> 0
